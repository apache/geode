userobjects

Pivotal GemFire Native Client C++
C++ API Programming Example

Version 1.0, March 2006

The userobjects example illustrates usage of the C++ caching API for distribution and retrieval of custom defined user types. The purpose is to demonstrates the code required to derive from gemfire::Cacheable to define a type suitable for storage as a value in the cache. The derivation of a gemfire::CacheableKey type is illustrated as well. The example is located in the GemFire software installation under GemFireInstallDirectory/examples/userobjects.

Setting Up the Environment

On Linux: In every shell used to run this example application, set these environment variables:

export GFCPP=GemFireInstallDirectory

export PATH=$GFCPP/bin:$PATH

export LD_LIBRARY_PATH=$GFCPP/lib:$LD_LIBRARY_PATH

On Windows: The environment variables were set automatically during installation.

By default, this product ships configured for multicast resolution of membership. If the network environment does not allow multicast, you can configure GemFire for unicast. See the GemFire User's Guide for instructions on configuring TCP transport for membership and discovery. To view the GemFire manuals, the online C++ API documentation, and the complete set of examples, see GemFireInstallDirectory/docs/index.html.

Running userobjects

You can run userobjects by simply invoking the executable. There are no command line arguments. The process registers custom serializable types, then connects to the distributed system. It creates a cache and region, and creates a new AccountHistory and puts it in the cache. It will then fetch the AccountHistory from the cache and update it.

Example: Running the executable

./userobjects
